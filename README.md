### Как работает скрипт

Скрипт можно запускать без дополнительных аргументов, в таком случае он попросит пользователя вставить последовательности вектора и донора и пары рестриктаз, разделенные пробелом. Можно запустить с использованием аргументов:

- -h, --help            show this help message and exit
- -fv, --fasta-vector vector.fasta
                        Путь к FASTA-файлу с последовательностью векторной плазмиды
- -fd, --fasta-donor donor.fasta
                        Путь к FASTA-файлу с последовательностью донора
- -vr, --vector-restriction R1 R2
                        Две рестриктазы для вектора через пробел (например EcoRI XhoI)
- -dr, --donor-restriction R1 R2
                        Две рестриктазы для донора через пробел (например EcoRI XhoI)
- -o, --output output.fasta
                        Путь к выходному FASTA-файлу. Иначе вывод в консоль

Скрипт представляет из себя набор функций, каждую из которых я опишу ниже, чтобы было понятно, как все устроено. 

**Первый блок** отведен обработке входных данных
- `read_fasta` - помимо ввода последовательноти, я добавил возможность подавать программе FASTA файл плазмиды, так как часто они в таком виде и хранятся у биологов и удобно испольховать их так, без дополнительных манипуляций. Эта команда считывает FASTA файл в объект Seq() из Bio.Seq
- `get_arguments` - описывает аргументы, которые можно передать программе. Есть возможность сохранить полученную после клонирования плазмиду в FASTA формате через аргумент `-o`
- `get_data` - парсит аргументы. Если последовательность или пара рестриктаз не были переданы через аргумент, пользователя попросят ввести их.

**Второй блок** - непосредственно анализ и клонирование

- `class Restrictase` - в дальнейшем для проверки рестриктаз мне будет нужен индекс позиции их сайта рестрикции. Чтобы не обращаться к ферменту и позиции через менее понятные индексы (если бы, например, я хранил их в кортежах вида (рестриктаза, индекс сайта)), я определил этот класс.
- `check_restrictase` - проверяет, есть ли в соответсвующей плазмиде сайт данной рестриктазы
- `check_all_restrictases` - проводит проверку с помощью функции `check_restrictase` для всех рестриктаз и создает объекты класса Restrictase с позициями их сайтов рестрикции. Если у какой-то из рестриктаз кол-во сайтов не равно 1 (либо нет вообще, либо их несколько), то программа выводит ошибку. Вместе с тем предварительно выводятся сообщения о сайтах для каждой рестриктазы, чтобы сразу можно было понять, есть ли проблемы с какой-то еще рестриктазой.
- `define_left_and_right_restrictase` - для того, чтобы определить, получится ли лигировать концы от таких пар рестриктаз, необходимо сопоставить рестриктазы, режущие c 5' ....
