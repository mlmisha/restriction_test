## Анализ совместимости рестриктаз для клонирования ## 

### Что есть что

- Основной скрипт хранится в `restr_test.py`.
- Зависиомсти в `requirements.txt`.
- Пример последовательности вектора в `vector.fasta`, пример последовательности донора в `donor.fasta`.

Дополнительно я сделал пример программы в колабе для запуска в формате ColabFold'а https://colab.research.google.com/gist/mlmisha/383db407760306a69f0c52db87b34d3e/test_gene.ipynb

### Как работает скрипт

Скрипт можно запускать без дополнительных аргументов, в таком случае он попросит пользователя вставить последовательности вектора и донора и пары рестриктаз, разделенные пробелом. Можно запустить с использованием аргументов:

- -h, --help            show this help message and exit
- -fv, --fasta-vector vector.fasta
                        Путь к FASTA-файлу с последовательностью векторной плазмиды
- -fd, --fasta-donor donor.fasta
                        Путь к FASTA-файлу с последовательностью донора
- -vr, --vector-restriction R1 R2
                        Две рестриктазы для вектора через пробел (например EcoRI XhoI)
- -dr, --donor-restriction R1 R2
                        Две рестриктазы для донора через пробел (например EcoRI XhoI)
- -o, --output output.fasta
                        Путь к выходному FASTA-файлу. Иначе вывод в консоль

```
python3 restr_test.py

#или с аргументами
python3 restr_test.py -fv vector.fasta -fd donor.fasta -vr EcoRI XhoI -dr EcoRI XhoI
```

Скрипт представляет из себя набор функций, каждую из которых я опишу ниже, чтобы было понятно, как все устроено. 

**Первый блок** отведен обработке входных данных
- `read_fasta` - помимо ввода последовательноти, я добавил возможность подавать программе FASTA файл плазмиды, так как часто они в таком виде и хранятся у биологов и удобно испольховать их так, без дополнительных манипуляций. Эта команда считывает FASTA файл в объект Seq() из Bio.Seq
- `get_arguments` - описывает аргументы, которые можно передать программе. Есть возможность сохранить полученную после клонирования плазмиду в FASTA формате через аргумент `-o`
- `get_data` - парсит аргументы. Если последовательность или пара рестриктаз не были переданы через аргумент, пользователя попросят ввести их.

**Второй блок** - непосредственно анализ и клонирование

- `class Restrictase` - в дальнейшем для проверки рестриктаз мне будет нужен индекс позиции их сайта рестрикции. Чтобы не обращаться к ферменту и позиции через менее понятные индексы (если бы, например, я хранил их в кортежах вида (рестриктаза, индекс сайта)), я определил этот класс.
- `check_restrictase` - проверяет, есть ли в соответсвующей плазмиде сайт данной рестриктазы
- `check_all_restrictases` - проводит проверку с помощью функции `check_restrictase` для всех рестриктаз и создает объекты класса Restrictase с позициями их сайтов рестрикции. Если у какой-то из рестриктаз кол-во сайтов не равно 1 (либо нет вообще, либо их несколько), то программа выводит ошибку. Вместе с тем предварительно выводятся сообщения о сайтах для каждой рестриктазы, чтобы сразу можно было понять, есть ли проблемы с какой-то еще рестриктазой.
- `define_left_and_right_restrictase` - перед описанием этой функции хочется обозначить некоторую проблему в постановке задачи (хотя, возможно, я неправильно понял). Необходимо сохранить ориентацию вставки из донора при клонировании в вектор. Однако в текущей постановке задачи неизвестно, какой именно отрезок плазмиды вставки вырезается и какой именно отрезок донора вставляется, так как после двух разрезов в каждом случае появляется два фрагмента. Из-за этого непонятно, какие сайты рестриктаз должны совпадать для оценки успешности лигирования. Здесь я решил воспользоваться общим представлением, что **обычно** вырезаемый из вектора фрагмент меньше остальной плазмиды. И, аналогично, фрагмент всавки из донора меньше, всей остальной его плазмиды. Первое допущение более обосновано, так как сайты рестрикции часто собраны в MCS. Второе более слабое, но в рамках этого задания я посчитал, что можно остановиться на этом. В дальнейшем, полагаю, можно просить пользователя указывать, что именно является вставкой. Итак, **что же делает функция?** В функции сравниваются расстояния между двумя сайтами рестрикции, определенными ранее. Сначала определяется, какой сайт находится ближе к началу строки последовательности и потом рассчитывается расстояние с учетом этого и того, что последовательность закольцована. По тому, какое расстояние получилось меньше определяются границы вырезаемого фрагмента вектора и вставки донора - определяются левые и правые сайты рестриктаз.  
- `check_compatibility` - для успешного лигирования нужно, чтобы друг другу подходили левые рестриктазы (соответсвующие началу вырезаемого фрагмента из вектора и началу вставки из донора) и отдельно правые. Первым делом с помощью функции `define_left_and_right_restrictase` определяются левые и правые рестриктазы. Далее проводится дополнительный контроль расстояния между ними. Если сайты расположены слишком близко друг другу (меньше 5 нт) рестрикция будет неэффективна. Рестриктазе нужно до 5 нт (для разных по-разному, но не больше 5 нт в любом случае) upstream сайта рестрикции, поэтому и межжду сайтами должно быть достаточно места. Если все нормально проводится оценка совместимости рестриктаз с помощью оператора `%`, который в для объектов Bio.Restriction определяет, можно ли лигировать концы, полученные сравниваемыми рестриктазами. В это входят и изошизомеры, и изокаудомеры и рестриктазы, образующие тупые концы. Далее я добавил еще одну проверку. Есть некоторая вероятность, что выбранные рестриктазы будут совместимы не только по одному из сайтов, но и по другому тоже - в таком случае есть опасность самолигирования и встравивания вставки в неправильной ориентации. Для этого я провожу дополнительную проверку, в случае если по основным сайтам все нормально. Если оказывается, что концы в векторе после расщепления  подходят друг к другу, то выводится предупрждение, что это чревато получением неправильных конструкций и лучше выбрать другие нуклеазы.
- `clone` - проводит клонирование по выбранным сайтам. Сначала выделяется последовательность вставки, потом удаляется фрагмент вектора и вставка с вектором соединяются. В результате получается последовательность, начинающаяся со вставки и продолжающаяся вектором. Так как у нас кольцевые молекулы - это неважно.
- `process` - последовательно запускает все описанные выше функции и возвращает результат
- `main` - для запуска скрипта. Сбор входных данных и запуск `process`. Выводит результат в консоль или, если есть флажок `-o` в указанный файл в FASTA.
